# Architecture Decision Records (ADR)

## 철학
- **Simplicity over Complexity**: 빌드 도구 없이 브라우저에서 즉시 동작하는 Vanilla JS 기반의 가벼운 설계.
- **Immediate Feedback**: 사용자의 모든 코드 변경이 즉시 결과로 반영되는 실시간성 강조.
- **Safety First**: 신뢰할 수 없는 코드 실행을 위해 브라우저의 전용 샌드박스 기능을 활용.

---

### ADR-001: 빌드 도구 없는 Vanilla JavaScript 기반 개발
**결정**: 초기 프로토타입 단계에서 React, Vite 등의 빌드 시스템을 배제하고 순수 HTML/CSS/JS 및 CDN 라이브러리(Monaco)를 사용.
**이유**: 설정 복잡도를 낮추고, 사용자가 생성된 코드를 즉시 이해하고 조작하기 가장 익숙한 환경을 제공하기 위함.
**트레이드오프**: 프로젝트 규모가 커질 경우 파일 관리 및 상태 동기화 로직이 복잡해질 수 있음. (향후 확장을 고려해 모듈형 JS 구조 사용)

### ADR-002: Monaco Editor CDN 통합
**결정**: 자체 에디터 구현 대신 VS Code 기반의 Monaco Editor를 CDN(unpkg)을 통해 로드하여 사용.
**이유**: 신뢰성 있는 구문 강조(Syntax Highlighting), 자동 완성, 다중 에디터 상태 관리 기능을 빠르게 확보하기 위함.
**트레이드오프**: 오프라인 환경에서 작동하지 않으며, 라이브러리 로딩 시간에 의존함.

### ADR-003: Iframe `srcdoc`을 이용한 실시간 미리보기
**결정**: 사용자가 작성한 HTML/CSS/JS를 하나의 문자열로 합쳐 `<iframe>`의 `srcdoc` 속성에 직접 삽입.
**이유**: 별도의 서버 사이드 렌더링이나 로컬 파일 저장 없이 클라이언트 측에서만 완벽하게 실시간 동작을 구현할 수 있는 가장 효율적인 방식.
**트레이드오프**: 큰 규모의 페이지일 경우 문자열 조작 비용이 발생할 수 있으나, 단일 페이지 빌더로서는 충분한 성능을 발휘함.

### ADR-004: Blob API 기반 클라이언트 사이드 다운로드
**결정**: 서버 요청 없이 브라우저의 Blob API를 활용해 `index.html`, `style.css`, `script.js` 파일을 물리적 파일로 변환하여 다운로드 처리.
**이유**: 서버 리소스를 사용하지 않으며 사용자 개인 정보를 보호하고 즉각적인 결과물을 제공함.
**트레이드오프**: 파일 개수가 많아질 경우 관리의 번거로움이 있을 수 있어 차후 ZIP 압축 기능 검토 예정.
